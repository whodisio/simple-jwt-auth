**what changed**: package-lock.json deleted, pnpm-lock.yaml added, packageManager field set to pnpm@10.24.0

**why it WILL break**: `.github/workflows/.test.yml` uses:
- line 36: `npm ci --ignore-scripts` — expects package-lock.json, will fail
- line 33: `cache: 'npm'` — caches npm, not pnpm
- all test steps use `npm run ...` — this still works, but install fails first

**required fix**: update CI workflow to use pnpm:
```yaml
- name: enable corepack
  run: corepack enable

- name: install
  run: pnpm install --frozen-lockfile
```

**how to verify**:
- update `.github/workflows/.test.yml` to use pnpm
- push and watch CI
